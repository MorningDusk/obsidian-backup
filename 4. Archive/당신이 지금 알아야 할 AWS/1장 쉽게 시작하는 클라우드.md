# 클라우드 컴퓨팅
> [!info] 클라우드 컴퓨팅 (Cloud Computing)
> 컴퓨터의 각종 연산 및 처리를 개인 PC가 아닌 인터넷 공간(클라우드)에서 처리하는 방식
- ex: 네이버 N 드라이브, MS 원드라이브, 구글 드라이브, 드롭박스 등
## 클라우드 컴퓨팅의 장점
1. 유연한 IT 인프라 관리 및 서비스 구축 가능
2. 신속한 인프라 도입
3. 수월해진 글로벌 서비스 제공
4. '오토 스케일링'을 사용하여 트래픽 폭주 대응 가능
5. 빅데이터, 인공지능 서비스의 한계점 극복
## 클라우드 컴퓨팅 유형
### 웹 호스팅 (Web Hosting)
- 하나의 서버에 여러 사용자를 수용할 수 있고, 사용자간 상호 접근은 격리되며, 코드를 올려 사용자가 원하는 애플리케이션을 작동 가능
- 다양한 오픈 소스를 자유롭게 활용하기에는 한계가 있어서 PaaS로 진화
### 온프레미스 (On-premise)
- 기업이 자체적으로 데이터 센터를 보유하고 시스템 구축부터 운영까지 모두 수행하는 형태
- 초기 시스템 투자에 드는 비용 부담이 크고, 시스템 가동 후 운영에 드는 비용도 시스템 사용량과 상관없이 일정 금액을 부담
## 클라우드 서비스 운용 형태
### 공용 클라우드 (Public Cloud)
- 인터넷을 통해 서버 및 저장소와 같은 컴퓨팅 리소스를 제공하는 타사 클라우드 서비스 공급자가 소유하고 운영
- ex: AWS, MS Azure
### 사설 클라우드 (Private Cloud)
- 단일 비즈니스 또는 조직에서 독점적으로 사용되는 클라우드 컴퓨팅 리소스
- 서비스와 인프라가 개인 네트워크 환경에서 유지 및 관리
### 하이브리드 클라우드 (Hybrid Cloud)
- 공용 클라우드와 사설 클라우드 사이 데이터와 응용 프로그램을 공유할 수 있는 기술
- 두 클라우드 간에 이동할 수 있도록 만들어져 더 높은 유연성과 개발 옵션 제공
## 클라우드 컴퓨팅 서비스 유형
클라우드 서비스는 어떤 자원을 제공하느냐에 따라 3가지로 구분
### IaaS (Infrastructure as a Service)
- 서비스로서의 인프라
- 네트워킹 기능, 컴퓨터 및 데이터 스토리지 공간을 제공
- IT 리소스에 대해 가장 높은 수준의 유연성과 관리 제어를 제공
- ex: 넷플릭스(Netflix)
### PaaS (Platform as a Service)
- 서비스로서의 플랫폼
- 소프트웨어 서비스를 개발할 때 필요한 플랫폼을 제공하는 서비스
- 사용자는 PaaS에서 필요한 서비스를 선택해 애플리케이션 개발하고, PaaS 운영 업체는 개발자가 소프트웨어를 개발할 때 필요한 API를 제공해 개발자가 좀 더 편하게 개발할 수 있게 지원
- 기본 인프라를 관리할 필요가 없어 애플리케이션 개발과 관리에 집중 가능
### SaaS (Software as a Service)
- 서비스로서의 소프트웨어
- 클라우드 환경에서 운영되는 애플리케이션 서비스
- 소프트웨어를 구입해서 PC에 설치하지 않아도 웹에서 소프트웨어를 빌려 쓸 수 있는 환경
- 서비스가 어떻게 유지 관리되는지 또는 기본 인프라가 어떻게 관리되는지 고민할 필요 없음
![[Pasted image 20250201205923.png]]
### FaaS (Function as a Service)
- 클라우드 공급자의 플랫폼에서 완벽하게 실행하는 서버리스 코드로 배포
- 플랫폼으로 사용하면 서버 인프라를 관리할 필요가 없고 함수를 실행하는데 필요한 컴퓨팅 작동 시간만큼 비용을 지불
### CaaS (Container as a Service)
- 클라우드 공급자에게 가상 컨테이너로 마이크로서비스를 빌드하고 배포
- 경량의 가상 컨테이너에서 서비스를 배포
- 컨테이너의 구축과 배포, 모니터링과 확장을 종합한 도구도 제공

# 클라우드 네트워크 기본
![[일반적인_네트워크_구성.excalidraw|800]]

## 네트워크 장비
### 라우터 (Router)
- IP 주소를 기반으로 서로 다른 지역의 컴퓨터를 연결하는 기기
- 네트워크 간 데이터 전송을 위해 최적의 경로를 설정해주며, 데이터를 해당 경로를 따라 통신 가능하게 도와주는 인터넷 접속 장비
### 스위치 (Switch)
- 여러 컴퓨터를 연결하는 네트워크 분배기 역할
- 하나의 네트워크 라인에 여러 대의 컴퓨터에 랜 케이블을 분배하는 장비
### 방화벽 (FireWall)
- 네트워크의 보안 사고나 문제가 확대되는 것을 막고 격리하려는 것
- 내부의 네트워크를 보호하기 위해 외부에서의 불법적인 트래픽 유입을 막고 허가된 트래픽만을 허용하려는 적극적인 방어 대책
- 시스템 사용자의 외부 접속을 제한하거나 보안 상의 문제로 인한 외부인의 사용을 제한하는데 사용
### IP주소
- 인터넷 서버로 접속할 수 있는 컴퓨터의 고유한 주소
- 현재 IPv4를 사용 중이며 4그룹으로 나눠진 최대 12자리의 번호로 이루어져 있고 각 자리의 숫자는 0~255까지를 사용
### 도메인 (Domain)
- 사용자의 입장에서 쉽게 이용할 수 있도록 문자열로 나열된 주소
## 포트와 프로토콜
### 포트 (Port)
- 프로그램을 식별하는 번호
- 잘 알려진 포트(well-known port): 0번 ~ 1023번
- 등록된 포트(registered port): 1024번 ~ 49151번
- 동적 포트(dynamic port): 49152번 ~ 65535번
### 프로토콜 (Protocol)
- 네트워크 통신을 할 때 통신하기 위한 사전 약속
### TCP/IP (Transmission Control Protocol/Internet Protocol)
- 신뢰성이 높은 방식으로, 정보를 전달할 때 중간 유실이나 흐름 장애 시 재전송을 하는 방식
![[Pasted image 20250201215956.png|500]]
### OSI 참조 모델 (Open System Interconnection Reference Model)
- 시스템에 상관없이 각 시스템이 서로 연결될 수 있도록 만들어주는 통신 모델
- 데이터 전송을 위해서는 7단계를 거쳐서 전송
### HTTP (Hypertext Transfer Protocol)
- 인터넷상에서 데이터를 주고받기 위한 클라이언트 서버 모델(Client-server model)을 따르는 가장 일반적인 프로토콜
- TCP/IP 최상위 계층 애플리케이션 레벨 프로토콜에서 작동하며 어떤 종류의 데이터이든 간에 전송할 수 있도록 설계
## HTTP 기반 웹 서비스
1. 사용자가 브라우저 주소창에 'www.homepage.com'을 입력
2. DNS 서버를 통해 주소 값과 맵핑되어 있는 IP 주소를 찾음
3. DNS 서버로부터 받은 IP 주소를 전달받음
4. 전송 계층으로 데이터를 전달
5. 데이터를 네트워크 계층이 통신할 때 사용하는 '패킷'으로 분해
6. 분해한 데이터를 판별하기 위해 일련번호를 부여
7. 전송 계층의 데이터를 네트워크 계층으로 전달
8. 프로토콜이 아닌 물리적인 서버 간 데이터 통신이 이루어지며, 인터넷에 해당하는 부분
9. 도착한 데이터를 일련번호에 맞게 조립
10. 데이터를 애플리케이션 레이어로 전달
11. 다시 전달받은 데이터를 서버로 전달

# 클라우드 핵심 기술
## 가상 서버
- 하나의 물리적인 서버와 마찬가지로 개별 운영체제나 애플리케이션을 실행시킬 수 있음
- 서버를 논리적으로 나누어 CPU, 메모리 등의 자원을 할당
![[Pasted image 20250202192335.png]]
## 가상화 기술

> [!info] 가상화 (Virtualization)
> 물리적인 하드웨어 장치를 논리적인 객체로 추상화하는 것을 의미

![[Pasted image 20250202204349.png]]
### 호스트 가상화
- 하드웨어 위에 기본이 되는 호스트 운영체제를 설치하고, 호스트 운영체제에 가상화 SW를 설치
- 가상화 SW 위에 게스트 운영체제를 작동시키는 가상화
- 필요 이상으로 CPU나 메모리 사용이 증가하는 오버헤드 발생
### 하이퍼바이저 가상화
- 하이퍼바이저를 배치해 HW와 가상 환경을 제어
- 컴퓨팅 자원을 효율적으로 사용
- 환경별로 다른 운영체제가 작동하기 때문에 가상 환경을 사용하기 위한 오버헤드 발생
### 컨테이너 가상화
- 운영체제에 논리적인 영역(컨테이너)을 만들어 넣고 애플리케이션을 작동하는 데 필요한 라이브러리와 애플리케이션 등을 컨테이너 안에 넣어 개별 서버처럼 사용할 수 있게 한 것
- ex: OpenVZ, LXC, Lunix vServer, FreeBSD Jail, Solaris Zones, Docker 등
## 분산 처리 기술

> [!info] 클러스터링 (Clustering)
> 대량의 데이터를 분산 처리하는 방법으로 여러 개의 서버를 결합하여 하나의 컴퓨터처럼 보이게 만드는 기술
- ex: 아파치 하둡(Apache Hadoop), 스파크(Spark)
## 서버리스 (Serverless)
- 클라우드 제공자가 동적으로 가상 머신 자원의 할당을 관리해 미리 설정된 서버리스 코드를 실행시키는 기술
- 언제 어디서나 네트워크에 접근이 가능하며, 데이터 저장 및 다양한 기능을 제공해 시간과 비용을 효율적으로 관리 가능
- ex: AWS
## 로드 밸런싱 (Load Balancing)
- 둘 혹은 셋 이상의 중앙처리장치 혹은 저장 장치와 같은 컴퓨터 자원들의 작업을 나누는 것
- 여러 대의 서버가 분산 처리하여 서버의 부하량, 속돌 저하를 방지해 분산 처리
## 스케일
### 스케일 업 (Scale-Up)
- 서버 자체의 성능을 업그레이드하여 처리 능력을 향상시키는 방법
### 스케일 아웃 (Scale-Out)
- 물리적으로 서버를 늘려서 시스템을 확장하는 방법
### 오토 스케일 (Auto Scale)
- 서비스가 집중되었을 때 서버 CPU 및 메모리 사용량이 일정 사용량을 초과하면 자동으로 가상 서버 대수를 증감시키는 기능
