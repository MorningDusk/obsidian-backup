# AWS 클라우드 인프라

> [!info] 리전(Region)
> - AWS가 주요 지역에 설치해 운영하는 여러 개의 인프라
> - 각 리전은 완전히 분리되어 있으며 AWS 콘솔 상에서도 완전히 다른 리소스로 구분

![[Pasted image 20250202230725.png]]
## 가용 영역 (Availability Zone)
- 리전은 가용 영역의 집합, 가용 영역은 데이터 센터
- 각 가용 영역은 실제 물리적으로는 완전히 독립
![[Pasted image 20250202231110.png]]
- 하나의 리전은 여러 개의 가용 영역으로 이루어져 있음
	- 하나의 데이터 센터에서만 운영하고 있다면 재난이 발생했을 때 리전 전체가 영향을 받음
> [!info] 가용성 (Availability)
> 가용 영역을 여러 곳에 두고 운영한다면 하나의 가용 영역에 문제가 발생해도 단순히 변경만으로 가용 영역을 운영할 수 있는 특성
## 에지 로케이션 (Edge Location)
- CDN을 이루는 캐시 서버
- 리전에 속한 '리전 에지 캐시'를 통해 데이터 속도를 개선
1. 사용자가 브라우저를 통해 이미지를 요청
2. DNS 서버에 IP 주소를 요청
3. 에지 로케이션 주소를 라우팅
4. 전달받은 주소로 데이터를 요청
5. 사용자가 요청하는 파일이 저장되어 있다면 단계 생략하고 바로 9단계
6. 데이터가 없는 경우 서버에 데이터를 요청
7. 데이터를 에지 로케이션에 전달
8. 데이터가 도착하면 사용자에게 전달
### CDN (Content Delivery Network)
- 웹 콘텐츠를 전달하기 위한 최적화된 네트워크
- 웹 콘텐츠가 저장된 서버 외에 다른 서버에 콘텐츠를 캐시해두고 사용자와 가까운 위치에 있는 서버가 대신 전달하면, 접속량을 분산시키게 되어 안정적으로 서비스를 제공

# AWS 서비스
## 컴퓨팅 서비스
- 가상 서버를 중심으로 AWS의 핵심이 되는 서비스를 의미
### EC2 (Elastic Compute Cloud)
- 종량제 형태로 과금이 되는 가상 서버
- 컴퓨터 리소스에 대해 전반적 관리를 도와줌
- 개별 인스턴스에 대해 총체적인 권한을 받음
### 람다
- 특정한 이벤트를 통해 입력 인자가 발생하면 연산 과정으로 출력 인자를 바꾸는 이벤트 기반의 컴퓨팅 서비스
- 소스코드만 사용하여 인프라에 대한 관리가 필요하지 않음
### ECS
- 도커 컨테이너 서비스
### 라이트세일 (Lightsail)
- 완전 관리형 가상 서버 서비스
- 클릭 몇 번만으로 서버를 구축 가능
### 빈스톡 (Beanstalk)
- EC2의 설정을 매우 쉽게 돕는 서비스
- 서버의 런타임부터 운영체제 환경까지 관리
## 스토리지
- 파일을 보관하거나 관리하는 저장 장소를 의미
### S3
- 뛰어난 안정성과 보안이 완벽한 스토리지를 손쉽게 만들 수 있게 해줌
- 서비스 운영 시 생성되는 이미지, 동영상, 오디오 파일들을 저장할 수 있게 함
### 글래시어 (Glacier)
- 백업 데이터와 같은 저장된 리소스들의 사용 빈도가 낮은 경우에 적합한 서비스
- 저렴한 저장 비용을 자랑하지만, S3에 비해서 트래픽 요금이 다소 높음
### 스토리지 게이트웨이 (Storage Gateway)
- 기존 온프로미스 환경과 AWS를 연결해주는 게이트웨이 서비스
- 기존 데이터들을 클라우드 환경으로 옮기지 않고도 바로 적용이 가능
### EBS (Elastic Block Storage)
- EC2와 연결할 수 있는 저장 장치 서비스
- 언제든 EC2를 백업할 수 있고 문제가 생겼을 때 복원 가능
## 데이터베이스
- 체계화된 데이터들의 모임
### RDS
- 전통적인 관계형 데이터베이스(RDBMS)를 구축하는 서비스
- 높은 가용성을 자랑하며, 자동으로 백업과 복원을 지원
### 다이나모 DB (DynamoDB)
- No-SQL 기반의 완전 관리형 데이터베이스
- 백업이나 복원, 스케일링과 같은 관리 작업을 자동으로 수행
- 저렴한 비용
### 엘라스틱 캐시(Elasticache)
- 메모리에 데이터를 저장하여 빠르게 입출력이 가능해지게 해주는 서비스
### Red Shift
- 완전 관리형 SQL 데이터 웨어 하우스
- 대용량의 정형 데이터를 처리하는 데 사용
- 저렴한 가격에 유지 보수 및 운영 가능
## 네트워크
- 인터넷 연결과 내부 인터넷(인트라)에 대한 서비스 담당
### VPC (Virtual Private Cloud)
- '가상 네트워크망'(VPN, Virtual Private Network) 구축 지원 서비스
- 권한 있는 사용자들만 접속 가능
### Route53
- DNS(Domain Name System) 서비스
- 도메인을 AWS 리소스와 연결할 때 사용
### 클라우드 프론트 (CloudFront)
- 정적 콘텐츠들을 최적화할 수 있게 만드는 CDN(Content Delivery Network) 서비스
- 클라우드 에지 로케이션 서버를 이용하여 콘텐츠를 배포
## 인공지능
- 개발자가 처음부터 구현하기 어려운 자연어 처리 또는 이미지 분석 등에 대해 API를 제공하고 사용하기 쉽게 도움을 주는 서비스
### 폴리 (Polly)
- 음성 합성 서비스로, 텍스트를 음성으로 바꾸는 TTS(Text to Speech) 서비스
- 일상 언어와 비슷한 수준의 자연스러운 음성 파일 생성 가능
### 렉스 (Lex)
- 대화형 챗봇(Chatbot) 서비스
- 음성과 텍스트를 지원하고 자연어의 처리를 돕는 서비스
### 레코그니션 (Rekognition)
- 이미지 인식, 분석 서비스
- 이미지를 사전에 필터링해주는 데 많이 사용
## 데이터 분석
### 퀵사이트 (QuickSight)
- 태블로(Tableau)와 같은 데이터를 시각화하는 서비스
- 데이터를 분석하며 시각적인 인사이트 제공
### 아테나 (Athena)
- 서버리스 기반 SQL 쿼리 서비스
- 구축해야 할 서버가 필요하지 않고 S3에 저장된 데이터를 SQL문을 통해 원하는 결과를 얻을 수 있음
### 클라우드 서치 (Cloud Search)
- 완전 관리형 검색 서비스
- 클롤링된 텍스트나 패킷을 언어별로 처리해주고 사용자에 맞게 관련성을 찾아 원하는 데이터를 찾아줄 때 사용
### EMR
- 빅데이터 분석 플랫폼인 하둡 프레임워크를 다른 서비스들과 연동하여 쉽고 빠르게 사용할 수 있는 서비스
### 키네시스 (Kinesis)
- 실시간 데이터 처리를 위한 서비스
- 데이터베이스를 실시간으로 분석하여 처리하고 결과를 출력
## 개발자 도구
### 코드 빌드 (CodeBuild)
- 원격 빌드 서비스
- 소스크도를 컴파일하는 단계부터 테스트 및 배포까지 설정 가능
### 코드 커밋 (CodeCommit)
- 사설 Git 저장소 서비스
### 코드 디플로이 (CodeDeploy)
- 애플리케이션 배포를 자동화할 수 있게 지원하는 배포 서비스
### 코드 파이프라인 (CodePipeline)
- 코드 커밋부터 코드 디플로이까지 한번에 설정할 수 있도록 돕는 서비스
# AWS 커퓨팅 선택

## EC2
- 네트워크와 스토리지 그리고 서버나 가상화 등을 신경 쓸 필요 없음
- 운영체제와 애플리케이션은 직접 관리 필요
## 빈스톡과 ECS
- 보안이나 운영체제를 관리할 필요 없음
- ECS는 런타임을 직접 설정해야 함
- 빈스톡은 런타임을 선택할 수 있으며 추가적인 런타임 설치가 필요 없음
## 람다와 라이트세일
- 람다는 데이터 계층을 제공하지 않음
- 라이트세일은 어떠한 인프라 설정 없이 원하는 애플리케이션 서비스를 바로 작동 가능
